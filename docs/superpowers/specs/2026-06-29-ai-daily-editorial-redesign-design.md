# AI 日报编辑台式改版设计

日期：2026-06-29
状态：已确认
范围：AI 日报列表页、详情页及移动端适配

## 1. 定位与目标

AI 日报应呈现为中文 AI 行业编辑部制作的情报简报，而不是 SaaS 仪表盘。

设计目标：

- 高信息密度，但通过字号、栏宽、留白和分隔线保持舒适阅读。
- 主新闻拥有显著层级，其余新闻可以快速扫读。
- 依靠排版和阅读节奏建立品牌感，避免卡片堆叠。
- 桌面端采用主栏加侧栏，移动端采用经过重排的单栏。
- 明暗主题都保留编辑部式、克制、纸张感的视觉特征。

明确避免：

- 三列等宽卡片、Bento Grid。
- 紫蓝渐变和大量彩色标签。
- 每条内容使用边框卡片。
- 过度圆角、阴影和动效。

## 2. 已选视觉方向

采用“A · 编辑台式”。

核心视觉语言：

- 保留博客全局导航，日报区域使用独立报头。
- 背景接近纸张的暖中性色；暗色模式使用低饱和深灰，不使用纯黑。
- 正文沿用清晰的无衬线字体；报头和重要标题使用中文衬线字体栈。
- 暖红色作为唯一主强调色，用于栏目名、序号、当前目录项和链接状态。
- 细实线、双线报头和栏间竖线承担分组作用。
- 默认无卡片、无投影；只允许在必要的交互状态中使用轻微背景变化。
- 动效仅保留链接、目录和行项目的短暂颜色过渡，并尊重减少动态效果设置。

## 3. 信息架构

### 3.1 日报列表页

列表页从“卡片列表”改为“期刊归档页”：

1. 报头：AI 日报、英文副标、更新节奏。
2. 最新一期头版：日期、主新闻标题、摘要、期号、新闻数量和阅读时长。
3. 往期归档：按日期倒序的紧凑行列表，显示日期、当期主新闻和进入箭头。
4. 空状态：保留报头，以一段简洁编辑部文案说明暂无日报。

列表页不新增筛选、搜索、分页或订阅功能。

### 3.2 日报详情页

详情页顺序如下：

1. 返回日报入口。
2. 报头：AI 日报、日期、期号。
3. 桌面双栏：
   - 主栏：主新闻、分类信息流、日报尾注。
   - 右侧栏：本期速览、目录、前后期导航。
4. 主新闻：取“01 今日头条”的第一条新闻，显示栏目、标题、摘要、置信度和来源。
5. 分类信息流：按编号章节顺序展示，每条新闻使用序号、标题、正文和来源链接。
6. 尾注：自动生成时间或原文已有的结尾说明。

现有彩色“产品经理视角”四宫格删除。Markdown 中存在编号章节“08 产品经理视角”时，按普通编辑部栏目展示；不存在时不额外生成。

## 4. 组件边界

建议拆分为以下职责清晰的组件：

- `AiDailyMasthead`：日报报头、日期和期号。
- `AiDailyLeadStory`：最新一期或详情页主新闻。
- `AiDailyIssueList`：往期归档行列表。
- `AiDailySection`：一个编号分类及其新闻列表。
- `AiDailyStory`：单条新闻的序号、标题、正文和来源。
- `AiDailySidebar`：本期速览、目录和前后期导航。
- `AiDailyMobileSummary`：移动端的速览与横向目录。

组件只接收已整理的数据，不在渲染层解析原始 Markdown。

## 5. 内容解析与数据流

Markdown 继续作为唯一内容源，不要求补改历史日报。

解析层在现有元数据之外提供：

- `issueId`：由日期生成，格式为 `YYYYMMDD`，显示为“第 YYYYMMDD 期”。
- `sections`：仅收录标题匹配 `## NN ...` 的编号章节。
- `leadStory`：编号 01 章节中的第一条有序列表新闻。
- `storyCount`：所有编号章节的新闻总数。
- `sourceCount`：去重后的外部链接数量。
- `readingMinutes`：按中文字符和英文单词数量估算，最少 1 分钟。
- `previousIssue` 与 `nextIssue`：根据已排序日报列表计算。

每个章节的内容边界从当前编号二级标题开始，到下一个二级标题结束。未编号的 Raw Data Summary 等附加章节不进入编辑部信息流。

主新闻标题优先取第一处加粗文本；摘要取同一列表项中去除标题、置信度标记和裸链接后的正文；首个外部链接作为来源。若无法完整提取，则回退为该列表项的 Markdown 渲染，不阻断页面。

列表页从每期解析结果读取主新闻标题和摘要，因此不依赖当前较通用的 frontmatter `description`。

## 6. 响应式设计

### 6.1 桌面端

- 宽屏内容区最大宽度约 1120–1200px。
- 详情页主栏与右侧栏比例约为 3:1；右侧栏使用 sticky 定位。
- 正文控制行长，避免主栏因屏幕变宽而无限拉伸。
- 列表页最新一期与归档保持单一阅读轴，不使用多列卡片。

### 6.2 平板与手机

在小于桌面断点时执行独立重排：

- 详情页侧栏移到正文上方，转换为“情报数、板块数、阅读时长”三项速览。
- 目录转换为可横向滚动的章节导航，点击后滚动到对应锚点。
- 主新闻、分类和新闻均为单栏。
- 日报列表页保留报头和最新一期头版，往期归档改为“日期 + 标题 + 箭头”的紧凑行。
- 正文基准字号约 16px，行高约 1.7；每行约 24–29 个汉字。
- 交互目标至少 44px 高；不依赖 hover 才能发现操作。
- 长 URL 省略显示但保留完整可点击地址。
- 表格、代码块和超宽媒体允许局部横向滚动，不撑破页面。
- 页面尊重安全区内边距。

主要验收宽度：390px、768px、1440px。

## 7. 交互与可访问性

- 当前目录项随滚动章节更新，并保持可见的文字与颜色状态。
- 点击目录使用平滑滚动；启用减少动态效果时改为即时定位。
- 所有链接保留清晰的键盘焦点样式。
- 暖红强调色在明暗主题下均满足正文与交互状态的可读对比度。
- 日期使用语义化 `time` 元素，主内容使用 `article`，目录使用带标签的 `nav`。
- 视觉序号不替代真实标题层级。

## 8. 异常与兼容处理

- 日报不存在：显示报头、未找到说明和返回日报入口。
- 日报列表为空：显示编辑部式空状态，不使用虚线卡片。
- Markdown 缺少编号章节：使用克制的编辑部正文样式渲染原文。
- 主新闻解析不完整：保留原始 Markdown 内容，避免丢失新闻。
- 历史内容中的未编号原始数据章节不进入目录和主信息流。
- 保留现有路由和 Markdown 文件格式，生成脚本无需因视觉改版而修改。

## 9. 验收与测试

实现完成后需验证：

- `npm run build` 和 `npm run lint` 通过。
- 日报列表页与现有全部历史日报均可访问。
- 主新闻、章节、目录、来源链接、期号和前后期导航正确。
- 390px、768px、1440px 下无横向页面溢出。
- 移动端目录可触控滚动，锚点定位不被 sticky 导航遮挡。
- 明暗主题的排版层级、分隔线和强调色清晰。
- 键盘焦点、语义标题顺序和减少动态效果设置有效。
- 不再出现三列卡片、彩色标签集合或产品经理四宫格。

## 10. 非目标

本次不改动：

- AI 日报抓取与生成策略。
- Markdown frontmatter 格式。
- 博客其他频道的视觉样式。
- 全站导航结构。
- 订阅、搜索、筛选、个性化推荐和数据分析功能。
