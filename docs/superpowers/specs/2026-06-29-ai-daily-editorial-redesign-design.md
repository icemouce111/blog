# AI 日报编辑台式改版设计

日期：2026-06-29
状态：已确认
范围：AI 日报列表页、详情页、移动端适配、历史格式对齐及全球 AI 应用趋势洞察

## 1. 定位与目标

AI 日报应呈现为中文 AI 行业编辑部制作的情报简报，而不是 SaaS 仪表盘。

设计目标：

- 高信息密度，但通过字号、栏宽、留白和分隔线保持舒适阅读。
- 主新闻拥有显著层级，其余新闻可以快速扫读。
- 依靠排版和阅读节奏建立品牌感，避免卡片堆叠。
- 桌面端采用主栏加侧栏，移动端采用经过重排的单栏。
- 明暗主题都保留编辑部式、克制、纸张感的视觉特征。

明确避免：

- 三列等宽卡片、Bento Grid。
- 紫蓝渐变和大量彩色标签。
- 每条内容使用边框卡片。
- 过度圆角、阴影和动效。

## 2. 已选视觉方向

采用“A · 编辑台式”。

核心视觉语言：

- 保留博客全局导航，日报区域使用独立报头。
- 背景接近纸张的暖中性色；暗色模式使用低饱和深灰，不使用纯黑。
- 正文沿用清晰的无衬线字体；报头和重要标题使用中文衬线字体栈。
- 暖红色作为唯一主强调色，用于栏目名、序号、当前目录项和链接状态。
- 细实线、双线报头和栏间竖线承担分组作用。
- 默认无卡片、无投影；只允许在必要的交互状态中使用轻微背景变化。
- 动效仅保留链接、目录和行项目的短暂颜色过渡，并尊重减少动态效果设置。

## 3. 信息架构

### 3.1 日报列表页

列表页从“卡片列表”改为“期刊归档页”：

1. 报头：AI 日报、英文副标、更新节奏。
2. 最新一期头版：保留原始期刊标题，例如“AI 日报 - 2026年06月28日”，再显示期号、主新闻标题、摘要、新闻数量和阅读时长。
3. 全球 AI 应用趋势：桌面端位于右侧栏，移动端位于最新一期之后，提供近周、近月、近年三个时间窗口。
4. 往期归档：按日期倒序的紧凑行列表，显示日期、当期主新闻和进入箭头。
5. 空状态：保留报头，以一段简洁编辑部文案说明暂无日报。

列表页不新增筛选、搜索、分页或订阅功能。

### 3.2 全球 AI 应用趋势

趋势模块是编辑部式信息栏，不使用图表仪表盘、卡片网格、关键词云或彩色标签。

- 近周、近月、近年各展示 3 条洞察。
- 每条洞察包含标题、一段简洁判断和可展开的来源链接。
- 模块显示更新时间、实际覆盖范围和来源数量。
- 桌面端默认显示近周，使用文字下划线切换时间窗口。
- 移动端紧跟最新一期，切换在当前页面完成，不新增路由。
- 原始期刊标题和主新闻标题均保留，趋势模块不得替代或弱化它们。

### 3.3 日报详情页

详情页顺序如下：

1. 返回日报入口。
2. 报头：AI 日报、日期、期号。
3. 桌面双栏：
   - 主栏：主新闻、分类信息流、日报尾注。
   - 右侧栏：本期速览、目录、前后期导航。
4. 主新闻：取“01 今日头条”的第一条新闻，显示栏目、标题、摘要、置信度和来源。
5. 分类信息流：按编号章节顺序展示，每条新闻使用序号、标题、正文和来源链接。
6. 尾注：自动生成时间或原文已有的结尾说明。

现有彩色“产品经理视角”四宫格删除。Markdown 中存在编号章节“08 产品经理视角”时，按普通编辑部栏目展示；不存在时不额外生成。

## 4. 组件边界

建议拆分为以下职责清晰的组件：

- `AiDailyMasthead`：日报报头、日期和期号。
- `AiDailyLeadStory`：最新一期或详情页主新闻。
- `AiDailyIssueList`：往期归档行列表。
- `AiTrendInsights`：近周、近月、近年趋势切换与来源列表。
- `AiDailySection`：一个编号分类及其新闻列表。
- `AiDailyStory`：单条新闻的序号、标题、正文和来源。
- `AiDailySidebar`：本期速览、目录和前后期导航。
- `AiDailyMobileSummary`：移动端的速览与横向目录。

组件只接收已整理的数据，不在渲染层解析原始 Markdown。

## 5. 内容解析与数据流

Markdown 继续作为唯一内容源。已具备编辑部栏目结构的历史日报保留原文；格式异常的历史日报按第 6 节规范化。

解析层在现有元数据之外提供：

- `issueId`：由日期生成，格式为 `YYYYMMDD`，显示为“第 YYYYMMDD 期”。
- `sections`：仅收录标题匹配 `## NN ...` 的编号章节。
- `leadStory`：编号 01 章节中的第一条有序列表新闻。
- `storyCount`：所有编号章节的新闻总数。
- `sourceCount`：去重后的外部链接数量。
- `readingMinutes`：按中文字符和英文单词数量估算，最少 1 分钟。
- `previousIssue` 与 `nextIssue`：根据已排序日报列表计算。

每个章节的内容边界从当前编号二级标题开始，到下一个二级标题结束。未编号的 Raw Data Summary 等附加章节不进入编辑部信息流。

主新闻标题优先取第一处加粗文本；摘要取同一列表项中去除标题、置信度标记和裸链接后的正文；首个外部链接作为来源。若无法完整提取，则回退为该列表项的 Markdown 渲染，不阻断页面。

列表页从每期解析结果读取主新闻标题和摘要，因此不依赖当前较通用的 frontmatter `description`。

日报模块加载时只接受 slug 匹配 `YYYY-MM-DD` 的 Markdown 文件，必须排除 `TEMPLATE.md` 和其他非日期文件。

### 5.1 趋势数据

趋势数据存放在独立的 `src/data/ai-trends.json`，避免把跨期聚合结果混入某一篇日报。

每个时间窗口包含：

- `window`：`week`、`month` 或 `year`。
- `rangeStart` 与 `rangeEnd`：实际覆盖日期。
- `updatedAt`：最近成功更新时间。
- `coverageCount`：参与分析的日报数量。
- `mode`：`curated` 或 `generated`。
- `insights`：3 条趋势洞察；每条包含标题、摘要和来源 URL 列表。

初始周、月、年内容通过可信全球资料研究后写入，必须保留直接来源链接和资料日期，不得使用占位文案。

后续由日报生成流程逐步更新：

- 近周：历史中至少有 4 期有效日报时允许自动生成。
- 近月：历史中至少有 14 期有效日报时允许自动生成。
- 近年：历史中至少有 60 期有效日报时允许自动生成。
- 自动生成只能引用参与分析的历史日报中已经存在的来源 URL。
- 输出包含未提供的 URL、少于 3 条洞察或无法解析时，视为失败，不覆盖上一次有效数据。
- 数据不足、LLM 不可用或更新失败时，继续展示最近一次可信内容，并明确显示其更新时间和实际覆盖范围。
- 周窗口最多每日刷新一次，月窗口最多每 7 天刷新一次，年窗口最多每 30 天刷新一次，避免无意义的重复调用。

## 6. 历史日报格式对齐

采用混合迁移，避免为了视觉统一而伪造历史编辑判断。

- 2026-06-21、2026-06-22、2026-06-24、2026-06-27、2026-06-28：保留现有编辑部栏目和正文内容。
- 2026-06-23、2026-06-25、2026-06-26：从 Raw Data Summary 或按来源转储的格式，机械整理为 `## 01 📡 原始信号归档`，并将不同数据源转换为三级标题。
- 原始信号归档保留已有条目、描述、指标和链接，不补写置信度、影响判断、机会或风险结论。
- 原始信号归档也使用报头、期号、信息流、分隔线和响应式布局；页面明确显示“原始信号归档”，不冒充完整编辑版日报。
- 历史 slug、日期、链接和 frontmatter 基础字段保持不变。

同步调整日报模板与生成器输出约定：

- 正常生成继续输出 `01–07` 编辑部栏目；已有或未来出现的 `08 产品经理视角` 作为可选栏目处理。
- LLM 分析失败时输出编号的“原始信号归档”，不得再输出 `Raw Data Summary` 或未编号的来源二级标题。
- 此调整只规范最终 Markdown 结构，不改变抓取来源、排序、过滤或 AI 分析策略。
- 历史内容变化后重新生成 RSS，保留原有条目链接。

## 7. 响应式设计

### 7.1 桌面端

- 宽屏内容区最大宽度约 1120–1200px。
- 详情页主栏与右侧栏比例约为 3:1；右侧栏使用 sticky 定位。
- 正文控制行长，避免主栏因屏幕变宽而无限拉伸。
- 列表页最新一期与归档保持单一阅读轴，不使用多列卡片。
- 列表页使用“最新一期与归档主栏 + 趋势右栏”；趋势右栏使用栏间竖线，不包裹成卡片。

### 7.2 平板与手机

在小于桌面断点时执行独立重排：

- 详情页侧栏移到正文上方，转换为“情报数、板块数、阅读时长”三项速览。
- 目录转换为可横向滚动的章节导航，点击后滚动到对应锚点。
- 主新闻、分类和新闻均为单栏。
- 日报列表页保留报头和最新一期头版，往期归档改为“日期 + 标题 + 箭头”的紧凑行。
- 趋势模块移到最新一期之后、往期归档之前，默认展开近周。
- 正文基准字号约 16px，行高约 1.7；每行约 24–29 个汉字。
- 交互目标至少 44px 高；不依赖 hover 才能发现操作。
- 长 URL 省略显示但保留完整可点击地址。
- 表格、代码块和超宽媒体允许局部横向滚动，不撑破页面。
- 页面尊重安全区内边距。

主要验收宽度：390px、768px、1440px。

## 8. 交互与可访问性

- 当前目录项随滚动章节更新，并保持可见的文字与颜色状态。
- 点击目录使用平滑滚动；启用减少动态效果时改为即时定位。
- 所有链接保留清晰的键盘焦点样式。
- 暖红强调色在明暗主题下均满足正文与交互状态的可读对比度。
- 日期使用语义化 `time` 元素，主内容使用 `article`，目录使用带标签的 `nav`。
- 视觉序号不替代真实标题层级。

## 9. 异常与兼容处理

- 日报不存在：显示报头、未找到说明和返回日报入口。
- 日报列表为空：显示编辑部式空状态，不使用虚线卡片。
- 趋势文件缺失或全部窗口无效：隐藏趋势栏，不显示空卡片。
- 单个趋势窗口无有效内容：禁用该窗口切换，并保留其他可用窗口。
- Markdown 缺少编号章节：使用克制的编辑部正文样式渲染原文。
- 主新闻解析不完整：保留原始 Markdown 内容，避免丢失新闻。
- 迁移后仍遇到未编号原始数据章节时，不将其加入目录和主信息流，并回退为编辑部正文样式。
- 保留现有路由与 frontmatter 基础字段；生成脚本仅调整最终 Markdown 的 fallback 结构。

## 10. 验收与测试

实现完成后需验证：

- `npm run build` 通过。
- 对本次新增和修改的 TypeScript/TSX 文件执行定向 ESLint 并通过；全量 `npm run lint` 不新增错误，同时记录当前 shadcn 组件与主题 hook 的 4 个既有基线错误。
- 日报列表页与现有全部历史日报均可访问。
- 主新闻、章节、目录、来源链接、期号和前后期导航正确。
- 2026-06-23、2026-06-25、2026-06-26 显示为原始信号归档，且内容、指标和链接无丢失。
- 正常模板与 fallback 模板均只产生可被页面解析的编号章节。
- `TEMPLATE.md` 不出现在日报归档、前后期导航或 RSS 中。
- RSS 重新生成后包含全部历史条目，slug 与链接保持不变。
- 趋势初始内容具有可访问的真实来源，周/月/年各 3 条。
- 趋势自动更新遵守 4/14/60 期阈值、刷新间隔和 URL 白名单，失败时不覆盖旧数据。
- 390px 下趋势模块位于最新一期之后，三个时间窗口可触控切换且无横向页面溢出。
- 390px、768px、1440px 下无横向页面溢出。
- 移动端目录可触控滚动，锚点定位不被 sticky 导航遮挡。
- 明暗主题的排版层级、分隔线和强调色清晰。
- 键盘焦点、语义标题顺序和减少动态效果设置有效。
- 不再出现三列卡片、彩色标签集合或产品经理四宫格。

## 11. 非目标

本次不改动：

- AI 日报的抓取来源、排序、过滤和分析策略。
- Markdown frontmatter 格式。
- 博客其他频道的视觉样式。
- 全站导航结构。
- 订阅、搜索、筛选、个性化推荐和数据分析功能。
